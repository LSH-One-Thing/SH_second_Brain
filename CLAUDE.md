# 나만의 지식 위키 — 운영 규칙 (Schema)

## 이 위키가 하는 일

인간의 비합리적 의사결정 구조를 분석하는 콘텐츠 제작자의 지식 아카이브.
행동경제학을 진입점으로, 심리학·진화심리학·신경과학·철학을 교차해
**하나의 원리로 여러 도메인을 동시에 푸는 사고**를 축적한다.

---

## 폴더 구조

```
One_thing_valut/
├── CLAUDE.md             # 이 파일 — 운영 규칙
└── One_thing/
    ├── raw/              # 원본 자료 (사용자가 추가, Claude는 읽기만)
    └── wiki/             # 정리된 지식 (Claude가 작성·관리)
        ├── index.md      # 전체 목차 (카테고리별 분류)
        └── log.md        # 작업 이력
```

- `raw/`와 `wiki/` 안에는 **하위 폴더 없이 플랫하게** 관리한다.
- 분류는 `wiki/index.md`에서 카테고리 섹션으로 처리한다.

---

## 운영 방법 3가지

### 1. 자료 넣기 (Input)

**흐름:** 사용자가 `raw/`에 파일 추가 → Claude에게 요청

**요청 방식:**
- `"raw 파일 처리해줘"` — raw/의 미처리 파일 전부 위키화
- `"[파일명] 위키로 만들어줘"` — 특정 파일 위키화
- `"[파일 내용] 이거 위키에 추가해줘"` — 텍스트 직접 붙여넣기

**Claude가 하는 일:**
1. raw 파일 읽기
2. 핵심 원리 추출 + 콘텐츠 방향성 관점에서 takeaway 작성
3. `wiki/[개념명].md` 생성 (callout 블록 → 본문 → 교차참조)
4. `wiki/index.md` 업데이트
5. `wiki/log.md`에 작업 기록

**위키 페이지 형식:**
```markdown
> **핵심 Takeaway (콘텐츠 방향성 관점)**
> [이 개념이 "인간 비합리적 의사결정 분석 콘텐츠"에서 어떻게 쓰이는지 2-3줄]

## [개념명]

[본문]

## 교차참조
- [[관련 개념 1]] — 연결 이유
- [[관련 개념 2]] — 연결 이유
```

---

### 2. 질문하기 (Query)

**흐름:** 사용자가 위키 내용에 대해 질문 → Claude가 wiki/ 파일들을 읽고 답변

**요청 방식 예시:**
- `"[[자의식-방어기제]]와 [[편도체-강탈]]의 관계 설명해줘"`
- `"손실 회피를 콘텐츠 후킹에 어떻게 쓸 수 있어?"`
- `"지금까지 정리된 개념 중 '자기 합리화'랑 연결되는 것 다 뽑아줘"`
- `"이 개념 [텍스트]을 위키 안에 어디에 연결시킬 수 있어?"`

**Claude가 하는 일:**
1. 관련 wiki 파일들을 읽어 교차참조 포함 답변 제공
2. 새로운 연결이 발견되면 기존 페이지에 교차참조 추가 제안

---

### 3. 건강검진 (Audit)

**흐름:** 사용자가 `"위키 건강검진 해줘"` 요청 → Claude가 전체 wiki 점검

**Claude가 확인하는 항목:**
1. **고아 페이지** — 다른 페이지에서 한 번도 참조 안 된 개념
2. **미처리 raw 파일** — raw/에 있지만 아직 위키화 안 된 파일 목록
3. **약한 연결** — 내용상 연결되어야 하는데 [[링크]] 없는 페이지 쌍
4. **중복 개념** — 같은 아이디어가 두 페이지에 나뉘어 있는 경우
5. **콘텐츠 방향성 정렬** — takeaway가 명확하지 않은 페이지

**결과물:** 건강검진 리포트 (개선 항목 목록 + 제안)

---

## 위키 작성 원칙

1. **Callout 블록 필수** — 모든 페이지 첫머리에 콘텐츠 방향성 관점 takeaway
2. **교차참조 적극 활용** — `[[페이지명]]` 형식, 페이지 하단에 교차참조 섹션
3. **원리 먼저, 예시 나중** — 추상 원리를 먼저 정의하고 구체 예시로 내려감
4. **반례 명시** — 원리의 적용 범위와 한계를 명확히 표시
5. **출처 추적** — 어떤 raw 파일에서 왔는지 페이지 하단에 기록

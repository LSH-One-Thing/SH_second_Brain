> **핵심 Takeaway (콘텐츠 방향성 관점)**
> AI와의 협업에서 "알아서 해줘"는 실패 공식이다. AI는 전체 맥락을 자의적으로 요약하고 기존 코드를 누락시킨다. **함수 단위로 쪼개고, 보존 영역을 선포하고, 구조 지도를 먼저 만드는 것**이 AI를 도구로 통제하는 원리다. 이것은 [[26.05.09 자청-AI-쪼개기전략|자청-AI-쪼개기전략]]의 "쪼개서 물어야 컨설턴트급 답이 나온다"와 같은 구조다.

## AI 코딩 협업 프로토콜

AI에게 코드 작업을 맡길 때 반복적으로 발생하는 "기존 코드 누락" 문제의 근본 원인과 해결 프로토콜.

### 왜 AI가 기존 코드를 날리는가

AI는 긴 코드를 생성하다가 앞뒤 문맥을 자의적으로 요약하거나 생략한다. "전체 코드 다시 짜줘"는 이 함정을 강제 발동시킨다. 해결 원칙: **AI를 수동적인 부품 공급자로 취급하고, 요청 단위를 최소화한다.**

---

## 프로토콜 3원칙

### ① 함수 단위 요청 (전체 코드 요청 금지)

❌ "전체 코드 다시 짜줘"
✅ "바뀌는 부분만 함수 단위로 줘"

예시 명령:
> "전체 코드는 건드리지 마. `BriefingPopup` 클래스랑, 그걸 실행하는 `on_universal_action` 함수 2개만 새로 작성해서 줘. 내가 알아서 붙여넣을게."

모듈(부품) 단위로만 요구하면 누락 사고가 0%가 된다.

### ② 보존 영역 선포

작업을 시킬 때, AI가 건드리면 안 되는 영역을 명시적으로 선언한다.

예시 명령:
> "기존 `load_excel_logic`이나 `config` 불러오는 부분은 절대 수정하거나 생략하지 말고 그대로 둬. 오직 팝업 로직만 추가해."

이 한마디가 AI의 "자의적 요약 알고리즘"에 브레이크를 건다.

### ③ 연결성 검증 강제

코드를 받으면 바로 복사하지 말고, 기존 코드와의 연결성을 먼저 확인한다.

> "이거 기존 변수(XPATHS_MAIN)랑 연결되는 거 맞아? 확인해."

AI가 "아차, 변수 선언부가 빠졌네요"라고 자수하게 만드는 구조.

---

## 코드 구조 시각화 5단계 (STEP A~E)

코드가 "돌아가긴 하지만 구조를 모르는 상태"를 해결하는 체계적 방법.

**STEP A — 전체 코드를 지도처럼 펼치기**
AI에게: "내 전체 코드를 섹션별로 지도처럼 나눠서 설명해줘. 각 섹션에서 어떤 일을 담당하는지 한 줄로 요약해줘."
→ 완전 이해가 목표가 아니다. 구조 파악이 목표다.

**STEP B — 각 섹션에 제목 달기**
코드 내에 직접 주석을 추가한다:
```
# 1. 데이터 읽기
# 2. 주소 바꾸는 로직
# 3. 결과 저장
```
함수/메서드 개념을 몰라도 된다. 중학생 수준의 언어로 달아도 된다.

**STEP C — 부분 코드 삽입 위치 자동 판단 시스템**
AI에게 부분 코드를 요청할 때 반드시 다음 3가지를 포함시킨다:
1. 이 코드가 들어가야 하는 섹션 이름
2. 내 기존 코드에서 어떤 줄 앞/뒤에 넣어야 하는지
3. 왜 그 위치인지 1문장 설명

예시 출력:
```
[섹션] 주소 변경 로직
[삽입 위치] line 84 바로 아래
[이유] 배송지 문자열 정규화 역할이므로, 실제 변경 함수 호출 직전에 위치해야 합니다.
```

**STEP D — 내가 위치를 다시 말로 확인**
> "그러니까 너가 준 부분 코드는 주소 정규화 기능이라, 기존 코드의 '배송지 변경 직전' 위치에 들어가야 하는 거 맞지?"
→ AI가 맞다고 하면 그 순간 머릿속 "전체 코드 지도"가 강화된다.

**STEP E — 1줄 단위 이해는 마지막에**
집을 다 지어놓고 나서 "이 벽돌이 왜 필요했지?"를 돌아보면 훨씬 빠르게 이해된다. 1줄 단위 이해는 구조 파악 이후에 한다.

---

## 핵심 원리

코드를 못 읽는 게 아니라 **코드의 위치, 영역, 역할이 머릿속에 지도가 없는 상태**가 문제다. 세 가지만 하면 해결된다:
1. 전체 코드를 섹션별로 나눠서 지도화
2. AI에게 부분 코드 줄 때 반드시 "삽입 위치"까지 설명하게 하기
3. 내가 위치를 다시 설명해보며 구조 이해 강화

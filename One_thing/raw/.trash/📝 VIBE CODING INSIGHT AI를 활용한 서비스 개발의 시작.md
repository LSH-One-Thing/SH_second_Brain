

### 1. 쉬운 프로젝트부터 시작해라

AI를 활용한 개발(바이브 코딩)의 성공을 위해 가장 먼저 해야 할 일은 **쉬운 프로젝트**를 통해 **프롬프트 작성법**과 **간단한 에러 해결 능력**을 익히는 것입니다.

#### 1-1. 쉬운 프로젝트의 정의 (정보 표시형)

- **정의:** 데이터를 생성하고 저장하는 기능 없이, 단순히 정보를 **보여주기만 하면 되는** 프로젝트입니다.
    
- **필요 요소:** **프론트엔드(Front-end)**만 필요합니다.
    
- **예시:**
    
    - 회사 소개 홈페이지
        
    - 개인 포트폴리오 웹사이트
        
    - 단순 정보성 웹사이트 (애플 웹사이트 등)
        

#### 1-2. 어려운 프로젝트의 정의 (데이터 저장/생성형)

- **정의:** 사용자가 **데이터를 생성하고 저장해야 하는** 기능이 포함된 프로젝트입니다.
    
- **필요 요소:** **프론트엔드**와 **백엔드(Back-end)**가 모두 필요하며, 관리 요소가 2배가 되어 난이도가 급격히 상승합니다.
    
    - _주요 난관:_ 프론트엔드와 백엔드 간의 **API 통신** 테스트 및 디버깅은 AI가 해결해주기 어렵습니다.
        
- **예시:**
    
    - SNS (블로그, 페이스북, 인스타그램 등)
        
    - 커머스 서비스 (쿠팡, 당근마켓 등)

### 2. 바이브 코딩을 위한 기본 지식 및 도구

#### 2-1. 필수 설치 도구

1. **Visual Studio Code (VS Code):** 코드를 작성하는 에디터(메모장 같은 개념)
    
2. **Cursor:** VS Code 위에서 작동하는 AI 툴 (AI 코딩 지원)
    
3. **Node.js:** 웹 개발 시 필수로 사용되는 자바스크립트 런타임 환경
    

#### 2-2. 프론트엔드의 세 가지 핵심 역할

쉬운 프로젝트를 위해 반드시 알아야 할 3가지 기본 요소입니다.

1. **HTML:** 웹사이트의 **구조**를 나타냄
    
2. **CSS:** 웹사이트의 **디자인/스타일** (색깔, 위치 등)을 담당
    
3. **JAVASCRIPT (JS):** 웹사이트의 **기능**을 담당
    

#### 2-3. 바이브 코딩 프롬프트 예시

- **핵심:** AI가 복잡한 방식으로 코드를 짜는 것을 방지하기 위해 **스펙을 디테일하게** 지정해야 합니다.
    
- **좋은 프롬프트 예시:**
    
    - `HTML, CSS, JAVASCRIPT를 이용해 메모앱 만들어줘.`

### 3. 디버깅 및 개발 노하우

#### 3-1. 에러 확인법

- **위치:** 웹페이지에서 오른쪽 클릭 > **검사(개발자 도구)** > **Console(콘솔) 탭**
    
- **정보:** 에러 발생 파일명과 정확한 줄 번호(`script.js`의 29번째 줄)를 알려주므로, 해당 내용을 복사하여 AI에 질문하면 해결이 쉽습니다.
    

#### 3-2. 디자인 요청 노하우

- 말로 설명하는 것보다 **디자인 이미지(사진 또는 피그마)를 첨부**하여 요청하는 것이 AI가 훨씬 정확하게 구현할 수 있습니다.
    

#### 3-3. 기능 추가 방법

- 여러 기능을 한 번에 요청하지 않고, **하나씩** 추가하는 것이 안정적입니다.
    
- 기능과 관련된 파일(Context)을 명확하게 지정(`이 파일 내에서만 수정해 줘`)하면 정확도를 높일 수 있습니다.

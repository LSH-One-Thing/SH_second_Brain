# 🐍 포기하지 않고 지식의 저주를 피하는 파이썬 문법 학습 가이드

## 🎯 학습 목표 및 현황

* **현재 단계:** 문법(Syntax) 공부 시작
* **극복할 두려움:** 지식의 저주(아는 것 같지만 모름), 학습 포기
* **핵심 전략:** 인출 연습 (Retrieval Practice) 극대화

---

## 1️⃣ 마인드셋 설정 (지식의 저주 방지)

* **☑️ '아는 것'과 '할 수 있는 것' 분리하기:** 강의를 끈 후 코드를 직접 칠 수 있어야 '할 수 있는' 상태로 인정합니다.
* **☑️ 완벽주의 피하기 (70%의 법칙):** 처음에는 70% 정도만 이해하고 넘어갑니다. 나머지는 프로젝트를 하면서 막힐 때 다시 돌아와 보충합니다.
* **☑️ 코딩 먼저, 이론 나중:** 이론을 외우기보다, 일단 강의 코드를 '따라 쳐서 실행'해보고 작동 원리를 역추적합니다.

---

## 2️⃣ 인출 연습 (Retrieval Practice) 극대화 전략

### 💡 방법 1: 빈칸 채우기 & 개념 설명 (설명력 테스트)

* **☑️ 핵심 문법 정리:** 오늘 배운 `for`, `if`, `def` 등의 핵심 구조를 빈칸 템플릿으로 만듭니다.
    * `for 변수 in 리스트: ______`
* **☑️ 강의 끄고 빈칸 채우기:** 책이나 강의 없이 빈칸을 떠올려서 채워 넣습니다.
* **☑️ 중학생에게 설명하기:** 배운 개념을 **가장 쉬운 언어로** 소리 내어 설명해봅니다. (ex: 함수는 마법 상자)

### 💡 방법 2: 코드를 거꾸로 읽기 (디버깅 능력 강화)

* **☑️ 예시 코드 선정:** 오늘 배운 잘 동작하는 코드 (5~10줄)를 준비합니다.
* **☑️ 역추적 실행:** 최종 결과(마지막 줄)부터 시작하여 1번 줄까지 거꾸로 올라가며 읽습니다.
* **질문:** "이 결과를 얻기 위해 바로 윗줄에서는 무슨 일이 일어났어야 했는가?"

### 💡 방법 3: 문법 챌린지 (미니 프로젝트)

새 문법을 배울 때마다 바로 적용할 수 있는 10분짜리 작은 목표를 만듭니다.

| 배운 문법      | 목표 (오늘 완료)                      | 완료 여부 |
| :--------- | :------------------------------ | :---- |
| Dictionary | 친구 3명의 이름/특징을 저장한 후, 내 정보 추가하기. | [ ]   |
| while 반복문  | 1부터 10까지 출력 중 7에서 멈추고 메시지 출력하기. | [ ]   |
| def 함수     | 숫자 두 개를 받아 뺄셈을 해주는 함수 만들기.      | [ ]   |
| ...        | ...                             | [ ]   |

## 3️⃣ 장기 기억 전략 (포기 방지 앵커)

* **☑️ 분산 학습 (Spaced Repetition) 스케줄:**
    * **오늘:** 새 내용 학습 및 인출 연습 10분.
    * **내일:** 어제 내용 인출 연습 5분.
    * **3일 뒤:** 지난주 내용 전체 훑어보기 5분.
* **☑️ 단기 세션 유지:** **매일 30분**처럼 짧고 굵게 자주 학습하여 지치지 않게 합니다.
* **☑️ 최종 프로젝트 앵커 설정:** 학습이 지루해질 때마다 이 목표를 떠올리며 동기 부여를 얻습니다.

> **나의 최종 프로젝트 (구체적 목표):**
> * *예시: 엑셀 데이터를 자동으로 읽어와서 특정 조건에 맞는 데이터만 필터링 해주는 프로그램 만들기.*
> * *오늘의 학습은 이 목표를 위한 중요한 '블록'이었다.*
